# DX21 Performance Mode Specification

## 目的

DX21 の Performance Mode として、`SINGLE` / `DUAL` / `SPLIT` の 3 モードをアプリへ組み込むための仕様を整理する。

本仕様は、添付された DX21 マニュアル抜粋の内容をもとに、UI 表示、発音仕様、音色選択、スプリット動作を実装向けにまとめたもの。

## モード概要

### SINGLE Mode

全鍵盤の音が同じ音色で発音されるモード。

- 1 つの音色を使用する
- ポリフォニック時は最大 8 音まで演奏可能
- 現在の通常演奏モードに相当する

### DUAL Mode

1 つの鍵盤を押すと、2 つの異なる音色が重なって発音されるモード。

- A 側音色と B 側音色の 2 音色を同時に使用する
- 1 キーにつき 2 音が鳴る
- ポリフォニック時は最大 4 音まで演奏可能
- 発音数の考え方は `2 音色 x 4 音`
- A 側/B 側の組み合わせを選択できる
- A-B 間のデチューン量を表示・設定対象にする

### SPLIT Mode

鍵盤上の任意のポイントを境に、左右で異なる音色を発音するモード。

- スプリットポイントより下側を A 側音色で発音する
- スプリットポイントより上側を B 側音色で発音する
- ポリフォニック時は左右それぞれ最大 4 音まで演奏可能
- ギター/ストリングスのような左右別音色演奏を想定
- スプリットポイントは MIDI key number で扱う

## LCD 表示仕様

LCD は 2 行表示を前提とする。

### SINGLE 表示

#### 表示形式

```text
PLAY SINGLE
A1   Vort Piano
```

#### 表示要素

- 1 行目: `PLAY SINGLE`
- 2 行目左: 音色番号
- 2 行目右: 音色名

#### 備考

マニュアル上は `SINGLE` キー押下で SINGLE モードになり、LCD にモード名と現在音色が表示される。

## DUAL 表示

### 表示形式

```text
DU: 2 ivoryEbony
16: 4 Folk Guit
```

### 表示要素

- 1 行目左: `DU`
- 1 行目中央: A 側の音色番号
- 1 行目右: A 側の音色名
- 2 行目左: A-B 間のデチューン量
- 2 行目中央: B 側の音色番号
- 2 行目右: B 側の音色名

### 音色選択

- A1-A16 から A 側音色を 1 つ選ぶ
- B1-B16 から B 側音色を 1 つ選ぶ
- A 側同士、または B 側同士の組み合わせはできない
- セレクトキーで音色選択を行う
- A1-A16 キーを押すと LCD 上段の表示が変わる
- B1-B16 キーを押すと LCD 下段の表示が変わる

### 発音仕様

- MIDI ノートオン 1 回につき A/B 両方を発音する
- A 側と B 側は同じ MIDI note を受ける
- B 側または A 側にデチューンを適用し、重なり感を作る
- 最大同時演奏数は 4 音

## SPLIT 表示

### 表示形式

```text
SP:10 Strings
60: 4 Folk Guit
```

### 表示要素

- 1 行目左: `SP`
- 1 行目中央: A 側、左側音色の音色番号
- 1 行目右: A 側、左側音色の音色名
- 2 行目左: スプリットポイント
- 2 行目中央: B 側、右側音色の音色番号
- 2 行目右: B 側、右側音色の音色名

### 音色選択

- スプリットポイントより下側の音色は A1-A16 から選ぶ
- スプリットポイントより上側の音色は B1-B16 から選ぶ
- セレクトキーで音色選択を行う

### スプリットポイント

- スプリットポイントは MIDI key number として扱う
- マニュアル表示例では `60`
- スプリットポイントより下側の音色の上限を示す
- SPLIT キーを押しながら任意の鍵盤を押すとスプリットポイントが変更される
- スプリットポイント変更時は LCD のスプリットポイント表示も更新する

### 発音仕様

- 入力ノートがスプリットポイント以下の場合は A 側音色を発音する
- 入力ノートがスプリットポイントより上の場合は B 側音色を発音する
- 左右それぞれ最大 4 音まで演奏可能

## 実装対象データ

Performance Mode 用に以下の状態を保持する。

```cpp
enum class PerformanceMode
{
    Single,
    Dual,
    Split
};

struct PerformanceState
{
    PerformanceMode mode = PerformanceMode::Single;
    int singleVoiceIndex = 0;
    int dualVoiceAIndex = 0;
    int dualVoiceBIndex = 0;
    int splitVoiceAIndex = 0;
    int splitVoiceBIndex = 0;
    int dualDetune = 0;
    int splitPoint = 60;
};
```

## 発音エンジン要件

### SINGLE

- 現在の `Dx21Engine` と同等
- 最大 8 voices
- 1 ノートにつき 1 パッチを発音

### DUAL

- A 側/B 側で別パッチを同時発音する
- 最大 4 voices
- 1 ノートオンに対して内部的には 2 voice を使用する
- ノートオフ時は A/B 両方を停止する
- デチューンを片側、または両側に適用できる構造にする

### SPLIT

- ノート番号で A/B どちらのパッチを使うか分岐する
- 最大 8 voices を左右 4 voices ずつに分ける
- ノートオフ時は、発音時に割り当てた側の voice を停止する
- スプリットポイント変更時、既存発音をどう扱うかは実装時に決める

## UI 要件

### 必要な操作

- Performance Mode 選択: `SINGLE` / `DUAL` / `SPLIT`
- A 側音色選択
- B 側音色選択
- DUAL デチューン設定
- SPLIT スプリットポイント設定

### LCD 更新

モード切り替え、音色選択、デチューン変更、スプリットポイント変更時に LCD 表示を更新する。

## 実装メモ

- まずは SINGLE を現在動作として扱い、PerformanceState を導入する
- DUAL/SPLIT は `Dx21Engine` を複数持つ方式、または Engine 内でパッチを複数扱う方式のどちらかを検討する
- UI だけ先に導入し、発音エンジン対応を段階的に進めることも可能
- 既存のファクトリーボイス一覧を A/B 選択に流用する
- LCD の 16 文字幅制限に合わせ、音色名は必要に応じて省略する
